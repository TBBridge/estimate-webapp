**版**: 1.0  
**作成日**: 2025-03-06  
**ステータス**: ドラフト（一問一答により取りまとめ）

## 1. 概要・目的

### 1.1 目的

登録済み代理店向けにアカウントを付与し、代理店ごとの仕切り率を反映した見積を作成する。見積データは指定の Excel テンプレートに自動入力し、PDF 化して申請用見積書として Web アプリ上に格納する。kintone（営業案件管理・ライセンス管理）および HubSpot と連携し、見積の承認・ダウンロード制御および案件管理・分析を可能にする。

### 1.2 スコープ

- 見積作成・編集・申請
- 見積書（Excel → PDF）の自動生成とストレージ
- kintone / HubSpot との顧客・案件データ連携（検索・新規作成・取得）
- 承認フローと通知（Teams / Slack / Gmail のいずれか 1 チャネルに統一）
- 承認後の見積書ダウンロード制御
- 自社管理者向け案件一覧・ダッシュボード分析
- 多言語対応（日本語・英語）

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 代理店 | 本アプリに登録され、見積を作成・申請する組織。自社管理者により手動登録される。 |
| 仕切り率 | 代理店ごとに適用されるマージン率。本製品用と保守料用で別マスタ。代理店 × 提供形態 × オプション（有無）で保持。 |
| 提供形態 | オンプレミス / サブスクリプション / クラウドのいずれか。 |

| 承認者 | 代理店ごとに固定 1 名。見積申請時に通知され、Web アプリ内で承認操作を行う。 |
| 自社管理者 | 代理店登録・マスタ管理・全案件の一覧・分析を行う運用側ロール。 |

## 3. 利用者・ロール

### 3.1 ロール一覧

| ロール | 説明 | 主な操作 |
|--------|------|----------|
| 自社管理者 | 代理店・マスタ・テンプレート・分析 | 代理店手動登録、仕切り率・製品単価・テンプレートマスタ管理、全案件一覧・ダッシュボード、見積書ダウンロード（承認後） |
| 代理店ユーザー | 見積作成・申請 | 見積作成（提供形態・契約形態選択、入力、計算）、kintone/HubSpot 連携トリガー、申請、承認後の自案件の見積書ダウンロード |
| 承認者 | 代理店に紐づく 1 名 | 承認待ち見積の一覧確認、Web アプリ内「承認」操作 |

### 3.2 アカウント・規模

- 代理店数: 約 30
- 1 代理店あたり 1 アカウント（代理店単位で 1 ログイン）
- 認証: メール + パスワードのみ（MFA・外部 IdP は対象外）

## 4. 機能要件

### 4.1 認証・アカウント

- 自社管理者が代理店を手動登録し、代理店用アカウント（メール・パスワード）を付与する。
- 代理店ごとに「承認者」を 1 名設定する（固定リスト）。
- ログイン後、ロールに応じてメニュー・操作範囲を制御する。

### 4.2 マスタ管理（自社管理者）

#### 4.2.1 代理店マスタ

- 代理店名、アカウント（メール）、承認者（1 名）、その他管理用項目。
- 仕切り率は別マスタで管理（4.2.2）。

#### 4.2.2 仕切り率マスタ

- **本製品用仕切り率**: 代理店 × 提供形態 × オプション（有無）で保持。随時変更可能。
- **保守料用仕切り率**: 代理店ごと（別項目・別マスタ）。随時変更可能。
- 決定・変更は自社管理者のみ。

#### 4.2.3 製品単価マスタ

- アプリ内で保持。提供形態別（オンプレ / サブスク / クラウド）で単価を管理。
- 自社管理者が登録・編集。

#### 4.2.4 見積書テンプレートマスタ

- **テンプレートファイル**: Excel ファイルを Web アプリにアップロードして保持。自社管理者が差し替え可能。
- **テンプレート識別**: 提供形態 × 契約形態の組み合わせごとに別ファイル。識別ルールは Web アプリ内マスタで管理。
- **セル割り当て**: テンプレート別に入力データのセル位置（例: B5=顧客名）をマスタで定義。初期は開発側で固定し、将来は画面上で登録・編集可能にすることも想定。
- 共通出力項目の例: 見積番号、日付、顧客名、有効期限 等。

### 4.3 見積作成フロー（代理店ユーザー）

#### 4.3.1 分岐制御

1. **提供形態**を選択（オンプレミス / サブスクリプション / クラウド）。
2. **契約形態**を選択（新規 / ライセンス追加 / オプション追加）。
3. 上記の組み合わせで、使用する **Excel テンプレート** および **入力項目** が決まる（テンプレートごとに入力すべき内容が異なる）。
4. 「ライセンス追加」「オプション追加」の場合は、**既存契約（親案件・既存見積）を画面上で選択して紐づける**。紐づけの正は HubSpot（取引）とする。

#### 4.3.2 入力内容

- **提供先顧客**: 都度、顧客名・住所等を手入力（顧客マスタからの検索・選択は行わない）。
- ライセンス数・オプション、その他テンプレート・計算に必要な項目を、分岐に応じて入力。

#### 4.3.3 金額計算

- 基本的に **ライセンス数** により金額が変動。
- 製品単価は提供形態別マスタを参照。
- 計算時に **代理店の仕切り率** を乗算。仕切り率は「代理店 × 提供形態 × オプション」で取得。
- **保守料**: 本体金額の 15% とオプション金額の 15% をそれぞれ計算。保守料にも代理店別の保守用仕切り率を適用。
- 契約形態（新規 / ライセンス追加 / オプション追加）により計算式・内訳が異なる場合は、仕様化時に計算ロジックを定義する。
- **補足**: Web アプリ画面には見積の計算式・内訳は表示しない（バックエンドで計算し、見積書 PDF にのみ反映する）。

#### 4.3.4 見積書生成・格納

1. 入力データを、選択されたテンプレートの **セル割り当てマスタ** に従い、該当 Excel テンプレートに自動入力。
2. Web アプリ上で **Excel → PDF 変換** を自動実行。
3. 生成した PDF を申請用見積書として Web アプリ上に格納（ストレージは実装時に決定）。

### 4.4 kintone 連携

- **連携アプリ**: 営業案件管理、ライセンス管理（2 アプリ）。
- **主なフィールド**: 客先企業名、ユーザー社名、保守開始日、保守終了日 等。
- **トリガー**: 見積作成・更新時に 1 回実行。
- **ロジック**:
  - 見積の提供先顧客情報（客先企業名・会社名 等）で検索。
  - 該当レコードが **存在しない** 場合: 新規作成。最低限、客先企業名・会社名を設定（その他は仕様・kintone 側制約に合わせて定義）。
  - **存在する** 場合: 必要なデータ（上記フィールド等）を取得し、見積画面や帳票で利用可能にする。

### 4.5 HubSpot 連携

- **連携オブジェクト**: 取引（Deal）等。取引内の **会社名** を顧客照合キーとして利用。
- **トリガー**: 見積作成・更新時に 1 回実行。
- **ロジック**:
  - 提供先顧客（会社名）で検索。
  - 該当レコードが **存在しない** 場合: 新規作成。最低限、会社名・都道府県・代理店・提供形態を設定。
  - **存在する** 場合: 必要なデータを取得。
- 「ライセンス追加」「オプション追加」時は、HubSpot の既存取引（親案件）を画面上で選択して紐づける。

### 4.6 承認フロー・通知

- 見積申請（申請用見積書を格納）すると、その代理店の **承認者 1 名** が承認待ちとして扱われる。
- **通知**: 承認者宛に、運用で選択した **1 チャネル** にのみ送信（全員同じ。Teams / Slack / Gmail のいずれか 1 つに統一）。
- **承認操作**: Web アプリ内の「承認」ボタンのみで完結。メール/Slack/Teams 上のリンクから承認する機能は不要。
- 承認済みになった見積は「承認済み」状態とし、ダウンロード制御に利用する。

### 4.7 見積書ダウンロード制御

- **ダウンロード可能なユーザー**:
  - その見積を依頼した **代理店スタッフ**（当該代理店アカウント）
  - **自社管理者**
- ダウンロード可能になるのは **承認後** のみ。承認前は不可。

### 4.8 案件管理・分析（自社管理者）

- **一覧**: 見積（案件）の一覧を表示。検索・フィルタ条件:
  - 代理店、期間、提供形態、契約形態、承認状態、顧客名
- **ダッシュボード**: 分析結果を画面で表示。
  - 代理店別: 見積件数、金額
  - 提供形態別・契約形態別の内訳
  - 時系列（月次推移等）
- 保存期間・ステータス（確定/失効等）の厳密なルールは特になし（必要に応じて後から定義）。

## 9. 提供形態×契約形態ごとの入力項目

提供形態・契約形態の組み合わせにより入力必須項目が異なる。共通として **提供先顧客**（顧客名・住所等）は都度手入力。

### 9.1 オンプレミス

| 契約形態 | 入力項目 |
|----------|----------|
| **新規** | i-Reporter ライセンス数、オプション有無（Web API / ConMas IoT standard 版 / ConMas IoT professional 版 / ConMas IoT MappingTOOL / i-Repo Link） |
| **ライセンス追加** | 既存ライセンス数、追加後ライセンス数、既存保守開始年月、既存保守終了年月、発注予定年月 |
| **オプション追加** | オプション有無（Web API / ConMas IoT standard 版 / ConMas IoT professional 版 / ConMas IoT MappingTOOL / i-Repo Link / i-Repo EdgeOCR（ライセンス数）/ i-Repo FreeDraw（ライセンス数）/ i-Repo WorkFlow（ライセンス数））、既存保守開始年月、既存保守終了年月、発注予定年月 |

### 9.2 サブスクリプション

| 契約形態 | 入力項目 |
|----------|----------|
| **新規** | ライセンス数、契約月数、オプション有無（Web API / ConMas IoT standard 版 / ConMas IoT professional 版 / ConMas IoT MappingTOOL / i-Repo Link） |

（サブスクリプションは新規のみ。追加はなし。）

### 9.3 クラウド

| 契約形態 | サブ種別 | 入力項目 |
|----------|----------|----------|
| **新規** | 年額 | i-Reporter ライセンス数、オプション有無（Web API / ConMas IoT standard 版 / ConMas IoT professional 版 / ConMas IoT MappingTOOL） |
| **新規** | 区切り | 上記に加え、区切り月数 |
| **追加** | ライセンスのみ | 既存ライセンス数、追加後ライセンス数、既存保守開始年月、既存保守終了年月、発注予定年月 |

（クラウドはオプション追加なし。）

## 13. 承認通知テンプレート

見積申請時に承認者へ送る通知文面（運用で選択した 1 チャネル: Teams / Slack / Gmail のいずれかで同一文面を使用）。

**件名（メールの場合）**

```
【見積承認依頼】見積番号：{見積番号} - {顧客名}
```

**本文**

```
見積の承認依頼があります。

■ 見積番号：{見積番号}
■ 顧客名：{顧客名}
■ 提供形態：{提供形態}
■ 契約形態：{契約形態}
■ 申請日時：{申請日時}
■ 申請者（代理店）：{代理店名}

以下のリンクから承認画面にアクセスし、承認または差し戻しを行ってください。

{承認画面URL}

※ 本メッセージは見積自動作成システムから自動送信されています。
```

Slack / Teams で送る場合は上記本文をメッセージ本文とし、件名は「見積承認依頼：{見積番号} - {顧客名}」等の短いタイトルとする。
